## A hierarquia, de fora para dentro

Pense em caixas dentro de caixas: **Região** contém **Zonas de Disponibilidade**,
que contêm **data centers**. As **Edge Locations** ficam fora dessa hierarquia,
espalhadas perto dos usuários.<sup><a href="#r1">1</a></sup>

<RegionAzMap />

## Os três conceitos

| Conceito                            | O que é                                                                                                  | Por que existe                                                                                                |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **Região (Region)**                 | Área geográfica nomeada (ex.: `sa-east-1` São Paulo, `us-east-1` N. Virginia). Contém múltiplas AZs.     | Escolher _onde no mundo_ seus dados/apps ficam: latência, custo e **conformidade legal**.                    |
| **Zona de Disponibilidade (AZ)**    | Um ou mais data centers isolados dentro de uma região. Uma região tem várias (normalmente 3+).           | **Alta disponibilidade:** se uma AZ cai (incêndio, enchente), as outras continuam. Você distribui entre AZs. |
| **Edge Location (borda)**           | Pontos de presença espalhados (bem mais numerosos que regiões), usados por CDN/CloudFront e DNS/Route 53. | **Baixa latência:** entrega conteúdo em cache _pertinho_ do usuário final, sem ir até a região.              |

<Callout variant="warn">
  **Distinção que cai muito:** _Região_ resolve **conformidade / onde os dados
  moram**. _Múltiplas AZs_ resolvem **alta disponibilidade / tolerância a falha**.
  _Edge Locations_ resolvem **latência de entrega ao usuário**. Casou o problema
  com o nível certo → acertou a questão.
</Callout>

## Como escolher uma Região (4 fatores)

A AWS lista quatro critérios — uma checklist perfeita para decisões de
liderança:<sup><a href="#r2">2</a></sup>

- ⚖️ **Conformidade** — leis que exigem dados no país (ex.: LGPD)
- ⚡ **Proximidade/latência** — perto dos usuários
- 🧩 **Serviços disponíveis** — nem todo serviço existe em toda região
- 💲 **Preço** — varia de região para região

<Callout variant="info">
  🧠 **Mnemônico:** **C-P-S-P** → **C**onformidade, **P**roximidade,
  **S**erviços, **P**reço.
</Callout>

## Conectando à sua missão

Num projeto da Caravela com clientes no Brasil, a resposta defensável é
normalmente **sa-east-1 (São Paulo)**: menor latência, e dados em solo brasileiro
ajudam na conformidade (LGPD). E para garantir que o sistema não caia se um data
center falhar, você espalha em **múltiplas AZs**. Isso é arquitetura básica de
produção — e agora você sabe justificar.
