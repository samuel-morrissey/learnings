Esta Aula exercita o **Catálogo v1**: a componentização 1:1 do vocabulário que as
Aulas já usavam em HTML. O Pedagogo escreve por _significado_ — `<MissionBox>`,
`<CompareCards>`, `<AskBox>` — e a Plataforma cuida do markup, do estilo e do
offline.

<MissionBox>
  Cada Componente do Catálogo encoda uma intenção pedagógica. Em vez de decidir
  _que HTML escrever_, o Pedagogo decide _o que ensinar_ — e o resultado fica
  on-brand porque todo Componente consome os mesmos design tokens.

  <span slot="meta">⏱️ ~6 min · pré-requisito: a Aula "Olá, MDX".</span>
</MissionBox>

## CompareCards — comparar N opções

`CompareCards` generaliza os antigos _triple cards_. É dirigido por dados, então
serve para duas, três ou mais opções, e uma pode ficar em destaque:

<CompareCards
  cards={[
    { tag: "EBS · block", label: "Bloco", description: "Um HD cru preso a uma instância." },
    { tag: "EFS · file", label: "Arquivo", description: "Um compartilhamento de rede que vários servidores montam." },
    { tag: "S3 · object", label: "Objeto", description: "Cada arquivo é um objeto acessado por uma key, via HTTP.", highlight: true },
  ]}
/>

<Callout variant="info">
  **Analogia de dev:** o destaque em "Objeto" mostra como `CompareCards` aponta a
  opção em foco sem que o Pedagogo escreva uma linha de CSS.
</Callout>

## Quiz — a island dirigida por dados

O `Quiz` é o primeiro Componente interativo. O Pedagogo passa as questões, as
alternativas, a resposta correta e a explicação — e nada de JS. A interatividade
fica escopada na island: feedback na hora ao responder e a pontuação ao final.

<Quiz
  questions={[
    {
      prompt: "Por que o Quiz é dirigido por dados?",
      options: [
        "Para o Pedagogo escrever JS na Aula",
        "Para o Pedagogo declarar só os dados (questões/resposta/explicação)",
        "Para abrir libs externas via CDN",
      ],
      answer: 1,
      explanation: "A interatividade vive escopada no Componente; a Aula só descreve o conteúdo.",
    },
    {
      prompt: "O que acontece ao responder uma questão?",
      options: [
        "A alternativa correta é destacada e o feedback aparece na hora",
        "A página recarrega",
        "Nada até o fim do Quiz",
      ],
      answer: 0,
      explanation: "E, ao final, a pontuação é exibida.",
    },
  ]}
/>

## AskBox — a tarefa de produção

A Aula fecha pedindo ao aluno que **produza** — é onde o aprendizado acontece:

<AskBox>
  Volte ao chat e descreva, **com suas palavras**, a diferença entre `bloco`,
  `arquivo` e `objeto`. Comprimir do seu jeito é o que fixa.
</AskBox>
