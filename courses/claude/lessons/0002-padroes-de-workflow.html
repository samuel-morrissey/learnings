Os 5 padrões de workflow

Workflow tem caminho fixo — você já sabe. Mas quais caminhos valem a pena? A Anthropic nomeia cinco. Saber escolher entre eles é metade das questões de cenário.

↩︎ Da Lição 01: num workflow, é o código que controla o caminho (não o modelo). Estes 5 padrões são os “caminhos de código” mais úteis. rever L01 · glossário
Por que importa: os cenários da prova (extração estruturada, pipeline de pesquisa, suporte) quase sempre têm um padrão “certo”. Reconhecê-lo na hora — e saber por que os outros não servem — é exatamente a competência de arquiteto que a missão pede.

O princípio antes dos padrões

Encontramos sucesso consistente usando padrões simples e componíveis, em vez de frameworks complexos. Comece com o mais simples possível; só aumente a complexidade quando ela claramente melhorar o resultado. — Anthropic, “Building Effective Agents” (paráfrase)

Os 5 são blocos componíveis: você combina vários num sistema real. Vamos do mais simples ao mais autônomo.

1

Encadeamento de prompts

prompt chaining

Decompõe a tarefa numa sequência fixa de passos; cada chamada LLM processa a saída da anterior.

Entrada→ LLM 1→ [gate?]→ LLM 2→ Saída
Use quando: a tarefa se divide em subpassos limpos e fixos. Trocar precisão por latência (mais passos, cada um mais fácil). Ex.: gerar um outline → escrever a partir dele. Pode ter um gate programático entre passos.
2

Roteamento

routing

Uma chamada classifica a entrada e a direciona para uma tarefa especializada de followup.

Classificador→ Rota A (ex.: reembolso) Rota B (ex.: técnico) Rota C (ex.: vendas)
Use quando: há categorias distintas de entrada, melhor tratadas separadamente, e a classificação é confiável. Permite prompts (ou modelos) especializados por rota. Ex.: triagem de tickets de suporte.
3

Paralelização

parallelization

Roda várias chamadas ao mesmo tempo e agrega. Dois sabores: sectioning (subtarefas independentes) e voting (mesma tarefa N vezes p/ diversidade/consenso).

Entrada→ LLM (parte 1) LLM (parte 2) LLM (parte 3) →Agregador
Use quando: subtarefas são independentes (ganha velocidade) ou você quer várias tentativas/visões para votar (ganha confiança). Ex.: revisar código por segurança, estilo e perf em paralelo.
4

Orquestrador-trabalhadores

orchestrator-workers

Um LLM central decompõe a tarefa dinamicamente, delega a LLMs trabalhadores e sintetiza os resultados.

Orquestrador→ Worker 1 Worker 2 Worker N (?) →Síntese
Use quando: você não sabe de antemão quantos/quais subtarefas existem — o orquestrador decide em runtime. Ex.: pesquisa multi-fonte, mudança de código em vários arquivos.
Nuance de prova

Paralelização vs. Orquestrador-trabalhadores: na paralelização as subtarefas são predefinidas por você; no orquestrador-trabalhadores é o modelo que define as subtarefas em tempo de execução. Por isso este padrão já beira o agente — é o degrau entre workflow e agente.

5

Avaliador-otimizador

evaluator-optimizer

Uma chamada gera uma resposta; outra avalia e dá feedback, num laço, até passar no critério.

Gerador→ Avaliador→ aprovado? → Saída ↺se não: refina
Use quando: existe um critério de qualidade claro e a crítica iterativa agrega valor mensurável. Ex.: tradução literária refinada, busca que precisa de várias rodadas. Atenção: é um laço — não confunda com agente. Aqui o laço é fixo (gerar↔avaliar); no agente, o modelo decide o que fazer no laço.

Heurística de seleção

PerguntaSe sim →
Passos fixos, cada um alimentando o próximo?Encadeamento
Entradas caem em categorias distintas?Roteamento
Subtarefas independentes (ou quer várias tentativas)?Paralelização
Subtarefas só conhecidas em runtime?Orquestrador-trabalhadores
Critério de qualidade claro + refino iterativo ajuda?Avaliador-otimizador
Nº de passos imprevisível, modelo dirige tudo?→ não é workflow: é agente (L03)

Teste rápido estilo prova

Identifique o padrão. Feedback na hora.

1. Tickets de suporte chegam misturados: reembolso, bug técnico e dúvida de vendas. Cada tipo tem um prompt e base de conhecimento próprios. Qual padrão estrutura a entrada do sistema?

2. Um sistema de pesquisa recebe um tema amplo. Quantas fontes consultar e quais sub-perguntas investigar só dá para saber depois de começar — varia por tema. Um LLM central decide as sub-perguntas e junta tudo. Qual padrão?

3. Para revisar um PR, você dispara simultaneamente três checagens fixas e sempre as mesmas — segurança, estilo e performance — e junta os comentários. Qual padrão?

4. Qual frase distingue corretamente paralelização de orquestrador-trabalhadores?
